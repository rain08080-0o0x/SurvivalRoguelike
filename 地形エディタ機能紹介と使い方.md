# 地形エディタ機能紹介と使い方

## 1. このファイルについて
このファイルは、`奈落塔地形エディタ` で現在使える機能と、基本的な使い方をまとめたガイドです。  
地形編集、配置物編集、保存と読込、バリデーション、プレイテストまでの流れを確認できます。

---

### 地形

- 頂点モードでの高さ編集
- 面モードでの高さ編集
- 複数頂点 / 複数面の一括編集
- セル属性の設定

  - 危険地形
  - 落下可
  - ロープアンカー
  - 削除済み面

- 頂点の削除と復活

### 配置物

- 採掘ポイントの追加 / 削除 / 編集
- プレイヤー開始地点の配置
- 帰還地点の配置

### マップ運用
- JSON 形式で保存 / 読込
- 新規作成
- 開く
- 別名保存

### 補助機能

## 3. 基本の流れ
1. `新規` または `開く` でマップを用意する
2. `レイヤー` ウィンドウで編集対象レイヤーを選ぶ
3. `地形編集対象` で `頂点` か `面` を選ぶ
4. 地形の高さやセル属性を編集する
5. ロープ、採掘ポイント、開始地点、帰還地点を配置する
6. `バリデーション` を確認して問題がないか見る
7. `保存` または `別名保存`
8. `プレイテスト起動` で本編確認する

## 6. 地形編集

### 6.1 頂点モード

#### できること
- 1頂点ずつ高さを上下する
- 複数頂点をまとめて高さ変更する
- 孤立頂点の削除
- 無効頂点の復活

#### 使い方
1. `地形編集対象` を `頂点` にする
2. 3Dビューで頂点をクリックして選択する
3. `選択頂点高さY` を変更するか、Gizmo をドラッグして上下する
4. 複数選択したい場合は `Shift + 左クリック`

#### 頂点削除の仕様
- 頂点削除は、周囲の面が先に削除されていて孤立した頂点だけ許可されます
- 四角形セル構造を壊す削除は拒否されます

#### 頂点復活の仕様
- `無効頂点を復活` で、現在レイヤーの無効頂点を復活できます
- 面を復活した時は、その面に必要な4頂点も自動で復活します

---

### 6.2 面モード

#### できること
- 1セル単位で面を上下する
- 複数セルをまとめて高さ変更する
- セル属性の設定
- 面の削除
- 削除済み面の復活

#### 使い方
1. `地形編集対象` を `面` にする
2. 3Dビューでセルをクリックして選択する
3. `面平均高さY` を変更するか、`面を+0.25m` / `面を-0.25m` を使う
4. 複数選択したい場合は `Shift + 左ドラッグ` または `Shift + 左クリック`

#### 面削除の仕様
- 面削除は、外周に接している面、または削除済み領域に連続する面だけ許可されます
- 内部の面だけを単独で抜いて、四角形セル構造が不自然になる削除は拒否されます

#### 面復活の仕様
- `選択面を復活` で、削除済み面だけ復活できます
- まだ削除されていない面を新規作成する機能ではありません

## 7. セル属性

### 設定できる属性
- `歩行不可`
  - 本編でそのセルへ入れなくなります
- `危険地形`
  - 本編で継続ダメージ対象になります
- `落下可`
  - 大きな下り段差を落下として許可しやすくします
- `ロープアンカー`
  - ロープ接続候補として扱います
- `削除済み面`
  - 床なし扱いになります

### 注意
- `削除済み面` のセルには床当たり判定がありません
- 側面コリジョンはありません。穴扱いです

## 17. 現時点の注意
- 面の復活は、削除済み面の復活のみです
- 頂点の復活は、無効頂点の復活のみです
- グリッド外へ新しい面や頂点を増やす機能ではありません
- 削除済み面には床当たり判定がありません
- 側面コリジョンはありません

## 小ステージエディタ 追加機能（2026-06-29）

- `File > Save / Load` から保存と読み込みを行えます。保存先や既存データの再編集をメニュー起点で統一しました。
- Save を開くと小ウィンドウが表示され、保存先区分やファイル名を確認して保存できます。編集中の内容を確定したい時は、先にこのウィンドウで保存先を確認してください。
- Load は Windows 標準ダイアログを使います。既存の小ステージJSONを選ぶと、その内容をエディタへ読み込みます。
- `View` メニューでは各 ImGui ウィンドウの表示 / 非表示を切り替えられます。作業内容に合わせて必要なウィンドウだけを表示してください。
- `小ステージHierarchy` には保存済み小ステージの一覧が表示されます。項目をクリックすると、その小ステージを読み込めます。
- Hierarchy の登録情報は `Assets/Naraku/Pieces/piece_hierarchy.cfg` へ自動保存されます。Hierarchy を更新した後は、この設定ファイルも変更対象として確認してください。
- 編集中のファイル名はメインウィンドウタイトルに表示されます。どの小ステージを編集中かを画面上部で確認できます。
- メニューバーにウィンドウが隠れないよう、初回表示位置は WorkPos 基準で少し下に補正されます。起動直後に上端へ張り付いて見える場合でも、従来より重なりにくくなっています。
